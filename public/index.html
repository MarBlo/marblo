Einiges gelernt zu Responsiveness und plot_ly

Für den zweiten Post wollte ich einige interacktive Elemente wie zum Beispiel DT::dataframe() oder plot_ly bzw ggplotlyeinsetzen, und bin dabei auf einige Überraschungen gestoßen - habe aber auch extrem viel gelernt.

Ausgangspunkt ist die phantastische Seite von Alison Presman Hills. Die Seite, aif der es um htmlwidgets geht, versuche ich im Folgenden nachzubauen.

Der englische Text ist der Originaltext, der so bleibt wie er ist, wenn alles wie erwartet geklappt hat. Probleme, Verbesserung oder Änderungen werden von mir auf Deutsch gemacht.

Das Setzen von global Options

Zu dem was von Alison vorgeschlagen wurde, habe ich noch widgetframe_isolate_widgets = TRUE hinzugefügt, das zu dem package widgetFrame gehört und später bei dem Einbinden von interaktiven Elementen noch eine große Rolle spielen wird. Der genaue Grund für diesen Befehl ist mir nicht klar und auch nach Lesen der Vignette nicht klar geworden.

Außerdem habe ich die Größe der in den codeChunks eingefügten Bilder eingestellt, obwohl Alison/Yihu schreiben, dass das nicht zu empfehlen ist. Ich habe aber gemerkt, dass es einfacher ist, die Größer der interaktiven Elemente in einem codeChunk einzufügen, wenn die chunk_opts etwas großzügiger gewählt werden. Man kann dann nämlich in dem codeChunk die Größe verkleinern, könnte sie aber nicht über das in den chunk_opts gemachten Angaben hinaus vergößern.

knitr::opts_chunk$set(comment = NA, tidy = FALSE, warning = FALSE, message = FALSE,
                      echo = TRUE, widgetframe_isolate_widgets = TRUE,
                      fig.width=6, fig.height=10)

Das wird hier in dem folgenden codeChunk schon angewendet, indem die fig.height aus den chunk_opts überschrieben wird. Würde man das nicht tun, würde zuviel Leerraum hinter dem Map entstehen.

  library(leaflet)
  library(widgetframe)
  l <- leaflet() %>% addTiles() %>% setView(0,0,1)
  frameWidget(l, width='90%')

frameWidget mit langem ggplot Code

Dient auch dazu, zu prüfen, wie lange die Seite lädt, wenn eine ganze Reihe an Sachen zu machen. Ich habe mittlerweile aber den dataFrame tttA in dem Projekt animation_RanglisteFahrer/diagrammeFahrer.R hergestellt und abgespeichert. Damit entfällt die Rechnerei in dem Chunk.

Schwierig ist noch die das Einlesen des .rds. Die Datensätze liegen jetzt direkt unter stm und können direkt mit readRDS('tttA') aufgerufen werden. Das beduetet aber, dass wenn der Post umzieht, zum Beispiel in Blog, dass dann auch die Datensätze mitgehen müssen.

Der Versuch, einen zentrale Datenordner einzurichten und den über here anzusprechen, ist in einem ersten Versuch gescheitert, muss aber laut rbind eigentlich gehen. Noch einmal lesen!

Auch hier funktioniert der interaktive Plot direkt nicht. Erst unter Zuhilfenehmen von frameWidget geht es wieder problemlos.

frameWidget(pp_ly, height = 1200, width = '100%')

Allerdings wird beim Übertrag in ein plotly-Objekt die Formatierung der facets verhauen. Bei einem Test der Responsiveness durch Verkleinern der Breite des Screens auf die Breite eines Handys erkennt man auch, dass die überschrift zu groß ist.

Da die Überschrift und vor allem die Facet-Titel so ohne weiteres als plotly-Objekt sich nicht verändern lassen, wird im Folgenden versucht, direkt mit plot_ly zu arbeiten.

nam <- tttA %>% group_by(Name) %>% distinct(Name) %>% `[[`(1)
pp_ly2 <- plot_ly(tttA %>% filter(Name == 'Simon Nuber'), hoverinfo ='skip') %>% 
  add_bars(x = ~week, y = ~Punkte, type ='bar', 
        text = ~paste0(Ort, '\n', Punkte), color = ~Ort, showlegend =FALSE, hoverinfo ='text') %>% 
  layout(barmode = 'stack', title = 'Simn', xaxis = list(title ='Woche', range=c(9,41)))

frameWidget(pp_ly2, height = 1200, width = '100%')

Funktioniert! Hier ist die Höhe des Plots auch nicht verändert und stellt somit die in opts_chunk definierten Größen dar.

Um das jetzt umzusetzen, müsste man den Plot aus dem plotly-Objekt wie bei ggplot als facet machen können. Das scheint aber nicht zu gehen. Stattdessen bietet plotly subplots an, die man flexibel auf Spalten und Reihen verteilen kann. Dazu müssen aber dann entsprechend mehrere einzelne Plots gemacht werden.

Will man die Top20 darstellen, muss man also 20 einzelne Plots machen, die man dann mit subplot arrangieren kann. Das macht man mit einer Funtion - und die hier unten folgende funktioniert tadellos.

nam <- tttA %>% group_by(Name) %>% distinct(Name) %>% `[[`(1)
plots <- lapply(as.character(nam[1:20]), function(x) {
  plot_ly(tttA %>% filter(Name == x), hoverinfo ='skip') %>% 
    add_bars(x = ~week, y = ~Punkte, type ='bar',
          text = ~paste0(Ort, '\n', Punkte), color = ~Ort, showlegend =FALSE, hoverinfo ='text') %>% 
    add_annotations(
    text = x,
    x = 0.02,
    y = 1,
    yref = "paper",
    xref = "paper",
    xanchor = "left",
    yanchor = "top",
    showarrow = FALSE,
    font = list(size = 12)
  ) %>% 
    layout(barmode = 'stack', 
           xaxis = list(title ='Woche', range=c(9,41), 
                        titlefont = list(size =12),
                        tickfont = list(size =9)),
           yaxis = list(title = 'Punkte',
                        titlefont = list(size = 12),
                        tickfont = list(size =9)),
           margin = list(r = 10, t = 25, b = 40, l = 10))
})
#htmltools::tagList(plots)
frameWidget(subplot(plots, nrows =10, shareX = TRUE, shareY = TRUE, margin = 0.01), height = 1200, width = '100%')
